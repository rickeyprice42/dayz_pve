# 🧟 DayZ PVE Server — Developer Guide

Простой и понятный гайд для работы с проектом. Если что-то сломалось — сначала проверь этот файл.

❗ Важно:
- В `DayZServer` запускается сервер

# 🚀 Первый запуск (один раз)

## 1. Клонировать репозиторий

```
git init
git pull origin main
ЛИБО
git clone https://github.com/rickeyprice42/dayz_pve.git (СКАЧАЕТ РЕПОЗИТОРИЙ В ОТДЕЛЬНУЮ ПАПКУ)

```

---

# 🔄 Как получать обновления

Каждый раз, когда есть изменения:

```
git pull origin main

```

ЛИБО:

👉 запусти `update_server.bat`

---

# ⬆️ Как загружать изменения (ВАЖНО)

Если ты что-то поменял:
```
git add .
git commit -m "описание изменений"
git push origin main
```

Пример:
```
git commit -m "changed loot balance"

```

---

# ⚠️ Правила работы

## ❌ НЕ ДЕЛАТЬ

- Не загружать моды (@папки) в git
- Не трогать `.gitignore`

## ✅ ДЕЛАТЬ

- Работать только в DAYZ SERVER
- После изменений делать commit + push
- Перед работой делать `git pull` (ЛИБО update_server.bat)

---

# 🔧 Автообновление сервера

Запуск:
```
update_server.bat
```
Что он делает:
1. Скачивает изменения
2. Копирует в сервер
3. (опционально) перезапускает сервер

---

# 🧠 Типичный workflow

## Ты (разработчик):

```
git add .
git commit -m "update"
git push
```

## Другой разработчик:

```
git pull
ЛИБО
update_server.bat
```

---

# 🆘 Если что-то не работает

## Ошибка при pull
👉 Решение:
```
git reset --hard
git pull origin main
```

---

## Конфликт файлов

👉 Лучше спросить, чем ломать

---

## Сервер не запускается

Проверить:
- правильно ли скопировались файлы
- нет ли ошибок в логах
